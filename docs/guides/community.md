# Community

Use GitHub for project discussion and support.

## Discussions

General questions, examples, and ideas belong in [GitHub Discussions](https://github.com/bniladridas/vesper/discussions).

Search first, then open a focused thread with the Ruby version, gem version, operating system, and a small code sample when relevant.

## Issues

Use [GitHub Issues](https://github.com/bniladridas/vesper/issues) for bugs and scoped feature requests.

For bugs, include the smallest reproduction you can share. For feature requests, describe the use case and the smallest API that would solve it.

## Pull Requests

Pull requests should stay narrow. Include tests for behavior changes and update docs when usage changes.

See [`Contributing`](../CONTRIBUTING.md) for setup and review notes.

## Security

Do not report vulnerabilities in public issues. Use [`GitHub Security Advisories`](https://github.com/bniladridas/vesper/security/advisories/new).
